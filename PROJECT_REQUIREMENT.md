Project Requirements: OpenEnv Real-World SimulationMeta PyTorch Hackathon — Round 11. Project ObjectiveThe goal is to build a complete, production-grade OpenEnv environment that simulates a genuine real-world task. This environment must allow an AI agent to learn and interact through a standardized API, moving beyond "toy" problems into utility-driven automation.2. Functional Requirements🛠️ Core Environment LogicReal-World Utility: The simulation must model a task humans actually perform (e.g., email triage, customer support, or data cleaning). Games and toys are strictly prohibited.OpenEnv Spec Compliance: * Implement the full interface: step(action), reset(), and state().Use Pydantic models for typed Observation, Action, and Reward.Include a valid openenv.yaml file for metadata.Baseline Inference: Provide an inference.py script in the root directory using the OpenAI Client to demonstrate the environment's solvability.📊 Task & Grading SystemTiered Difficulty: Minimum of 3 tasks (Easy, Medium, Hard).Deterministic Graders: Each task must have an automated grader that outputs a score between 0.0 and 1.0.Reward Shaping: The reward function must provide partial progress signals throughout the trajectory rather than a simple binary 1/0 at the end. It should also penalize "infinite loops" or destructive actions.3. Non-Functional Requirements📦 Deployment & DevOpsHugging Face Spaces: The environment must be deployed as a containerized Space tagged with openenv.Containerization: A working Dockerfile is mandatory. The environment must start via docker build and docker run without manual intervention.Infrastructure Limits:CPU/RAM: Must run on 2 vCPUs and 8GB Memory.Runtime: Inference script must complete in under 20 minutes.📝 DocumentationThe README.md must include:Environment description and motivation.Definitions of Action and Observation spaces.Task descriptions with expected difficulty levels.Baseline scores and setup instructions.4. Evaluation CriteriaParameterWeightDescriptionReal-world Utility30%Is this a genuine gap-filler for the RL/Agent community?Task & Grader Quality25%Are graders deterministic? Is the difficulty progression logical?Environment Design20%Clean state management and sensible reward shaping.Code Quality15%Spec compliance, passing openenv validate, and working Docker.Creativity10%Novelty of the problem domain and mechanics.5. Technical Constraints & VariablesThe following environment variables must be defined in the configuration:API_BASE_URL: Endpoint for the LLM.MODEL_NAME: Identifier for the inference model.HF_TOKEN: Hugging Face API key.LOCAL_IMAGE_NAME: (If using Docker image methods).📢 Mandatory Stdout FormatTo pass automated evaluation, inference.py must emit logs strictly in this format:[START] task=<name> env=<benchmark> model=<model>[STEP] step=<n> action=<str> reward=<0.00> done=<true/false> error=<msg/null>[END] success=<true/false> steps=<n> score=<score> rewards=<r1,r2...>6.
